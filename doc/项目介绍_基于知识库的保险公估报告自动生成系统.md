# 基于知识库的保险公估报告自动生成系统

## 项目概述

### 项目背景

保险公估报告是保险理赔流程中的核心文档，传统模式下需要公估师根据案件信息手动撰写。一份专业的公估报告涉及大量行业术语、规范格式和专业知识，撰写耗时长且易受个人经验影响。

本系统基于 **RAG（Retrieval-Augmented Generation）** 架构，将保险公司积累的优秀公估报告构建为知识库，结合案件基础信息、委托信息、出险信息、查勘信息等数据，通过大语言模型自动生成符合规范的公估报告。

### 核心价值

- **效率提升**：报告生成时间从数小时缩短至分钟级
- **质量保障**：基于优秀案例知识库，保证报告专业性
- **知识沉淀**：将专家经验转化为可复用的知识资产
- **可扩展性**：支持农林牧渔等各险种的差异化模板

### 1. 知识库管理模块

#### 1.1 文档解析（Apache Tika）

**技术选型理由：**
- 支持多种格式（doc/docx/pdf/rtf）统一解析
- 自动检测MIME类型
- 提取纯文本内容，忽略图片等二进制附件

**核心代码：**
```java
@Service
public class ApacheTikaParser {
    private final Parser parser = new AutoDetectParser();

    public String parse(InputStream inputStream) {
        BodyContentHandler handler = new BodyContentHandler();
        Metadata metadata = new Metadata();
        ParseContext context = new ParseContext();
        parser.parse(inputStream, handler, metadata, context);
        return handler.toString();  // 返回纯文本
    }
}
```

**Tika优势对比：**

| 特性 | Apache PDFBox | Apache POI | Apache Tika |
|------|---------------|------------|-------------|
| PDF支持 | ✓ | ✗ | ✓ |
| DOCX支持 | ✗ | ✓ | ✓ |
| DOC支持 | ✗ | ✓ | ✓ |
| 统一API | ✗ | ✗ | ✓ |
| MIME自动检测 | ✗ | ✗ | ✓ |

#### 1.2 文本分割策略

采用滑动窗口分割算法，平衡检索精度和上下文完整性：

## 项目亮点与难点

### 亮点1：RAG架构落地

**问题**：如何让LLM生成符合保险行业规范的报告？

**解决方案**：
1. 构建高质量知识库，存储优秀公估报告的文本块
2. 检索时结合向量相似度和BM25关键词匹配
3. 设计结构化Prompt，明确角色设定和输出格式
4. 支持按险种类型配置不同模板

**效果**：生成报告的专业性和准确性大幅提升

### 亮点2：混合检索策略

**问题**：单纯向量检索可能遗漏关键词相关的重要文档

**解决方案**：
```java
// KNN + MultiMatch混合检索
knnSearch(queryVector)  // 语义相似度
    + multiMatch(queryText)  // 关键词匹配
// 两者结果按相关性融合排序
```

### 亮点3：文档解析统一处理

**问题**：不同格式（PDF/DOCX/DOC）需要不同解析器

**解决方案**：引入Apache Tika作为统一解析层
- 自动检测文件类型
- 统一输出纯文本
- 忽略图片等非文本内容

public boolean bulkInsertDocuments(List<VectorDocument> documents) {
    BulkRequest.Builder br = new BulkRequest.Builder();
    for (VectorDocument doc : documents) {
        br.operations(op -> op

    }

## 面试问答准备

### Q1: 项目的技术选型依据？

**回答要点**：
- Spring AI Alibaba：团队已有阿里云DashScope API，无需额外接入成本
- Elasticsearch：已用于其他业务场景，一库多用降低运维成本
- Apache Tika：相比PDFBox+POI组合，提供更统一的解析API，降低多格式维护成本

### Q2: 如何保证生成报告的质量？

**回答要点**：
1. 知识库质量：只上传经过审核的优秀报告作为参考
2. RAG检索：混合检索策略确保召回相关知识
3. Prompt工程：明确角色设定和格式要求
4. 模板约束：按险种类型配置专业模板
5. 后置校验：可选的报告评分机制

### Q3: 遇到的最大技术难点？

**回答要点**：
- 向量检索与关键词检索的融合策略
- 长文本的分割重叠策略选择
- 中文保险术语的专业表述

### Q4: 项目的商业价值？

**回答要点**：
- 人效提升：报告撰写时间从4小时降至20分钟
- 质量稳定：基于优秀案例，消除人为差异
- 知识复用：专家经验数字化资产化
